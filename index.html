실전 경험 부족

실제 대화 상황이 오면 입이 얼어버린다. 일상에서 영어를 쓸 기회 자체가 없다.

→ 언제든 원하는 상황으로 연습 가능

💸

높은 접근 비용

원어민 과외는 1회 3~10만원. 꾸준히 이어가려면 수십만 원이 필요하다.

→ 무료 AI 모델로 비용 장벽 제거

😰

심리적 부담

실수에 대한 두려움, 발음에 대한 수치심. 영어를 쓰는 것 자체가 스트레스다.

→ AI 상대라 부담 없이 반복 연습

🔁

피드백 부재

혼자 연습하면 같은 실수를 반복한다. 잘못된 표현이 굳어버린다.

→ 대화 후 즉시 문법·표현 피드백

코드는 어떻게 정리되어 있나

lib/ ├─ constants/ # 색상·문자열·API 설정 ├─ models/ # Scenario, Feedback… ├─ providers/ # Riverpod 상태 ├─ services/ # 4개 Agent API 호출 ├─ screens/ # 8개 화면 │ ├─ auth/ │ ├─ scenario/ │ ├─ conversation/ │ ├─ feedback/ │ ├─ history/ │ └─ analysis/ ├─ widgets/ # 공통 위젯 ├─ router/ # go_router └─ main.dart

레이어 = 폴더

아키텍처 4계층이 디렉토리로 그대로 매핑된다. 새 기능 추가 시 위치가 명확하다.

services/ 분리 원칙

모든 AI 호출은 services/에서만. 화면(screens)은 절대 API를 직접 부르지 않는다.

기능별 화면 폴더

screens/ 아래 기능명으로 폴더를 나눠 팀원 간 충돌을 최소화했다.

개발자 관점에서 실제로 겪고 해결한 문제들이다.

❌ JSON 파싱 오류
AI가 응답을 마크다운 코드블록으로 감싸거나, maxOutputTokens 부족으로 JSON이 잘림.
✅ 코드블록 제거 정규식 + 토큰을 2048/4096으로 상향
❌ Flutter 웹 CORS
브라우저에서 API 직접 호출 시 CORS 차단.
✅ 네이티브 Gemini API(URL 키 포함) 방식으로 우회
❌ 백엔드 아키텍처 변경
초기 Supabase 설계 → MVP 범위 대비 과도하다고 판단.
✅ 서버리스 직접 호출로 전환, ADR-0003에 근거 기록
❌ 대화 맥락 유실
단발성 호출로는 이전 대화를 AI가 기억 못 함.
