영어가 두려운 사람이 AI와 대화하며 말하기 자신감을 키우는 앱

원하는 상황을 입력하면 AI가 시나리오를 만들고, 실제 영어로 대화하며, 끝나면 피드백을 준다. 비용 없이, 부담 없이, 언제든지.

01 — 문제 정의

영어 말하기,
왜 여전히 어려운가

한국인의 영어 말하기 실력이 낮은 원인은 단순하다 — 안전하게 반복 연습할 공간이 없다. MEF는 비용·심리적 부담·피드백 부재를 동시에 해결한다.

🧊

실전 경험 부족

단어와 문법을 알아도 실제 대화 상황이 오면 입이 얼어버린다. 일상에서 영어를 쓸 기회 자체가 없다.

→ 언제든 원하는 상황으로 연습 가능

💸

높은 접근 비용

원어민 화상 영어는 1회에 3~10만원. 꾸준히 이어가려면 한 달에 수십만 원이 필요하다.

→ 무료 AI 모델로 비용 장벽 제거

😰

심리적 부담

실수에 대한 두려움, 발음에 대한 수치심. 사람 앞에서 영어를 쓰는 것 자체가 스트레스다.

→ AI 상대라 부담 없이 반복 연습 가능

🔁

피드백 부재

혼자 연습하면 같은 실수를 반복하게 된다. 잘못된 표현이 굳어버리는 악순환이 이어진다.

→ 대화 후 문법·표현 피드백을 즉시 제공

03 — 솔루션

어떻게 해결하는가

MEF는 역할이 다른 4개의 AI Agent로 구성된다. 사용자가 입력한 상황 하나로 전체 학습 흐름이 자동으로 만들어진다.

"상황 입력 하나로
시나리오 → 대화 → 피드백까지 자동"

🎬 ScenarioAgent — 시나리오 생성

담당: 김령균 · 완료 ✅

한국어 상황 입력 → AI가 배경·역할·목표를 JSON으로 자동 생성.

입력: "카페 주문" → 바리스타 역할 시나리오

💬 ConversationAgent — 실시간 대화

담당: 이동희 · 완료 ✅

시나리오를 바탕으로 AI 파트너와 실시간 영어 대화 진행. 이전 대화 전체를 기억해 맥락 유지.

AI는 역할에 맞게 영어로만 응답

📋 FeedbackAgent — 즉각 피드백

담당: 이동희 · 13주차 예정

대화 종료 후 문법 오류, 더 자연스러운 표현, 자연스러움 점수(100점)를 제공.

예: "I am go" → "I go" / 점수: 78점

📊 AnalysisAgent 유료·14주차

담당: 김령균 · 14주차 예정

누적 대화 기록을 분석해 반복되는 실수 패턴을 개인화 리포트로 제공.

예: "3인칭 단수 's' 누락"이 70% 이상

04 — 아키텍처

Presentation Layer

화면 & 위젯

사용자가 보고 조작하는 UI. 데이터를 직접 처리하지 않고 상태만 반영.

Application Layer

상태 & 흐름 (Riverpod)

화면 간 데이터 공유와 로딩·성공·실패 상태 관리. 메모리에 저장.

Domain Layer

비즈니스 규칙 (Services)

Agent별 서비스 파일 분리. 프롬프트 설계, JSON 파싱, 오류 처리.

Data Layer

외부 데이터 (Google AI Studio)

HTTP POST로 Gemini API 호출. JSON 응답을 파싱해 Domain Layer로 전달.

05 — 기술 선택 근거 (ADR)

왜 이 기술을 선택했나

기술 선택의 이유를 ADR(Architecture Decision Record) 문서로 기록했다. 대안과 비교해 우리 프로젝트에 가장 맞는 것을 선택했다.

ADR-0001

플랫폼 선택

결정

Flutter

대안

React Native, Android(Kotlin), iOS(Swift)

이유

하나의 코드로 멀티플랫폼 지원. Dart가 초급자도 배우기 쉽고 빠른 프로토타이핑에 적합.

ADR-0002

상태 관리

결정

Riverpod

대안

Provider, BLoC, GetX

이유

Provider의 context 의존 문제 해결. AsyncValue로 로딩·성공·실패 3상태를 깔끔하게 처리.

ADR-0003

백엔드 & AI

결정

Google AI Studio (서버 없음)

대안

Supabase + Edge Function, Firebase, OpenRouter

이유

Gemini Flash 무료 1,500 req/일. 컨텍스트 100만 토큰으로 RAG에도 적합. 서버 없이 직접 호출.

07 — AI 활용 솔직성

AI가 만든 것 vs 우리가 한 것

이 프로젝트는 Claude Code(AI 코딩 어시스턴트)를 적극 활용했다. AI가 생성한 코드를 그냥 쓴 게 아니라, 이해하고 검토하고 수정하는 방식으로 개발했다.

🤖 AI(Claude Code)가 생성한 것

AI

디렉토리 구조 제안 및 초기 파일 생성 (lib/ 전체 뼈대)

AI

화면 위젯 초안 코드 (login_screen, scenario_input_screen 등)

AI

API 호출 로직 초안 (scenario_service.dart)

AI

ADR 문서 초안, 이 발표자료

✅ 우리가 검토·수정·결정한 것

검토

ScenarioAgent 프롬프트 설계 — 출력 형식과 역할 지시문을 직접 작성·수정

검토

JSON 파싱 오류 처리 — 마크다운 코드블록 제거 로직 직접 디버깅

결정

Supabase → OpenRouter → Google AI Studio 아키텍처 변경 결정

검토

모든 생성 코드를 직접 읽고 동작 방식 이해 후 병합

"AI가 코드를 생성했고, 저희는 왜 이렇게 동작하는지 이해하고, 프롬프트와 로직을 직접 수정했습니다. AI는 도구였고, 설계와 판단은 저희가 했습니다."
